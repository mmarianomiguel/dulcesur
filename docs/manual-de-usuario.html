1. Introducción al sistema

Este sistema es una plataforma integrada de ecommerce + punto de venta (POS) diseñada para negocios mayoristas y minoristas. Cuenta con dos interfaces principales:

  • Panel administrativo — Accesible desde /admin. Permite gestionar ventas, clientes, productos, stock, caja, compras, reportes y configuración.
  • Tienda online — Accesible desde la raíz /. Permite a los clientes finales navegar el catálogo, armar su carrito, y realizar pedidos.

Tecnologías utilizadas

BackendNext.js API Routes + Supabase (PostgreSQL)
AutenticaciónSupabase Auth
ImágenesCloudinary
PagosStripe
DeployVPS con webhook de auto-deploy

Convenciones de moneda

El sistema opera en pesos argentinos (ARS). Los montos se formatean con separador de miles punto y decimal coma (ej: $1.250,50). La zona horaria utilizada es America/Argentina/Buenos_Aires.

2. Configuración inicial

Esta sección describe los pasos necesarios para poner en marcha el sistema desde cero.

Crear un archivo .env en la raíz del proyecto con las siguientes variables:

Importante
Las variables marcadas como SECRETA nunca deben exponerse en el navegador. Solo las variables con prefijo NEXT_PUBLIC_ son accesibles desde el cliente.

2.2 Base de datos

El sistema usa Supabase (PostgreSQL). Para configurar la base de datos:

  1. Crear un nuevo proyecto en supabase.com.
  2. Ir al SQL Editor de Supabase.
  3. Ejecutar los scripts de migración en orden, que se encuentran en la carpeta /scripts:
    • migrate.sql — Esquema inicial (tablas, funciones, triggers)
    • migrate-v2.sql hasta migrate-v17-fix-hora-cast.sql — Migraciones incrementales
  4. Verificar que las tablas se crearon correctamente: empresa, usuarios, productos, clientes, ventas, tienda_config, etc.

2.3 Primer acceso

  1. Instalar dependencias: npm install
  2. Iniciar el servidor: npm run dev
  3. Crear el primer usuario administrador haciendo un POST a /api/usuarios con los datos: nombre, email, password, y es_admin: true.
  4. Acceder a /login con las credenciales creadas.
  5. Al ingresar, se redirige automáticamente al panel de administración en /admin.

2.4 Datos de la empresa

Ir a Configuración → Empresa para completar los datos del negocio:

  • Nombre comercial — El nombre visible del negocio
  • Razón social — Nombre legal/fiscal
  • CUIT — Clave Única de Identificación Tributaria
  • Situación IVA — Responsable Inscripto, Monotributista, Exento, etc.
  • Domicilio fiscal, teléfono
Consejo
Estos datos se usan en los recibos de venta, facturas, y en el pie de página de la tienda online. Completarlos primero evita tener que regenerar documentos después.

2.5 Personalización visual (White-Label)

Ir a Configuración → Apariencia para personalizar la identidad del sistema:

  • Nombre del sistema — Se muestra en la barra lateral del admin
  • Subtítulo — Texto secundario debajo del nombre
  • Logo — Subir imagen o ingresar URL. Se muestra en sidebar y tienda
  • Color primario — Elegir entre 8 presets (Azul, Verde, Rojo, Naranja, Rosa, Cyan, Violeta, Gris) o ajustar el tono con el slider
  • Oscuridad del sidebar — Controla qué tan oscura es la barra lateral

Los cambios se aplican en tiempo real y se sincronizan entre todas las pestañas del navegador.

Se pueden desactivar módulos que no se necesiten (ej: si no se usa Compras o Proveedores). Los módulos desactivados desaparecen del menú lateral. Dashboard y Configuración siempre están activos.

4. Ventas

El módulo de ventas es el corazón del sistema. Comprende el punto de venta, historial, documentos comerciales, y gestión de pedidos.

4.1 Punto de venta (POS)

Interfaz rápida para registrar ventas presenciales. Funcionalidades:

  • Búsqueda de productos — Por nombre, código o escaneando código de barras
  • Selección de cliente — Buscar por nombre, CUIT o crear cliente nuevo
  • Presentaciones — Seleccionar unidad (Unidad, Caja x12, Medio Cartón, etc.)
  • Conversión automática — Si el cliente pide 24 unidades y existe "Caja x24", el sistema sugiere la conversión
  • Descuentos — Aplicar descuento porcentual al total
  • Recargo por transferencia — Se aplica automáticamente si se configura
  • Medios de pago — Efectivo, Transferencia, Cuenta Corriente, Mixto
  • Impresión de recibo — Genera recibo térmico o PDF

Flujo de una venta

  1. Seleccionar el cliente (o dejar como "Consumidor Final").
  2. Agregar productos buscando por nombre o escaneando código de barras.
  3. Ajustar cantidades y presentaciones según lo solicitado.
  4. Aplicar descuento si corresponde.
  5. Seleccionar el medio de pago y completar datos (monto efectivo, cuenta bancaria, etc.).
  6. Click en Confirmar venta. El sistema descuenta stock, registra en caja, y genera el comprobante.
Pago Mixto
Si el cliente paga parte en efectivo y parte con transferencia, seleccionar "Mixto" e ingresar el monto de cada medio. El sistema valida que la suma coincida con el total.

Acciones por venta

  • Ver detalle — Ítems, montos, forma de pago
  • Editar — Modificar cantidades, precios, medio de pago, observación
  • Imprimir — Generar recibo PDF
  • Marcar entregada — Cambiar estado a "Entregada"
  • Crear Nota de Crédito — Desde la venta original
  • Anular — Revierte stock, saldo, y movimientos de caja
Anulación de ventas
Al anular una venta, el sistema automáticamente: devuelve el stock, revierte el saldo del cliente, revierte los movimientos de cuenta corriente, y genera un movimiento de cancelación en caja. Esta operación no se puede deshacer.

4.3 Carga manual

Formulario para registrar ventas de forma manual (ej: ventas hechas por fuera del POS, facturas de papel).

  • Permite seleccionar tipo de comprobante, número, fecha, cliente
  • Agregar ítems con descripción, cantidad, precio y descuento
  • Útil para migrar datos históricos o registrar operaciones externas

4.4 Hojas de ruta

Organiza las entregas del día en una vista por pestañas según el día de entrega.

  • Muestra: cliente, dirección, teléfono, total, estado de entrega
  • Permite confirmar entrega y cobrar en el momento
  • Enlace a Google Maps para navegación
  • Filtro por método de entrega (Envío / Retiro)

4.5 Notas de crédito

Registrar devoluciones de productos o ajustes a favor del cliente.

  1. Seleccionar el cliente.
  2. Vincular a la venta original (se cargan automáticamente los ítems).
  3. Ajustar las cantidades a devolver (no puede superar la cantidad original).
  4. Seleccionar método de devolución: Efectivo, Transferencia, o Cuenta Corriente.
  5. Confirmar. El sistema devuelve stock y ajusta el saldo del cliente.

4.6 Notas de débito

Registrar cargos adicionales al cliente (ej: intereses, fletes, ajustes).

  • Se puede vincular opcionalmente a una venta original
  • Los ítems son de libre descripción (no vinculados a productos)
  • Suma al saldo del cliente

4.7 Anticipos

Gestionar pagos anticipados de clientes.

  • Registrar anticipo con monto, concepto y cliente
  • Aplicar anticipo a una venta futura
  • Devolver anticipo si no se utiliza
  • Estados: Vigente Aplicado Devuelto

4.8 Remitos y facturación

Los remitos son comprobantes preliminares que luego se convierten en facturas.

Facturación en lote

Permite seleccionar múltiples remitos pendientes y convertirlos en facturas en una sola operación. Incluye barra de progreso y filtros por cliente y fecha.

4.9 Pedidos online

Gestionar los pedidos realizados desde la tienda online.

  • Ver pedidos pendientes con datos del cliente, dirección, y total
  • Confirmar entrega con cobro (efectivo, transferencia, cuenta corriente, mixto)
  • Imprimir recibo de entrega
  • Cancelar pedido con motivo
  • Generar nota de crédito por devoluciones

Estados: Pendiente Armado Confirmado Entregado Cancelado

5. Clientes

5.1 Gestión de clientes

Base de datos completa de clientes con toda la información comercial y fiscal.

Datos del cliente

Situaciones IVA soportadas

  • Responsable Inscripto → Factura A
  • Monotributista → Factura B o C
  • Exento → Factura B o C

Filtros de búsqueda

Buscar por nombre, CUIT, documento. Filtrar por: activo/inactivo, zona de entrega, vendedor asignado.

Importación masiva

Se pueden importar clientes desde un archivo Excel. También se puede exportar la lista completa a Excel.

5.2 Cuenta corriente

Cada cliente tiene una cuenta corriente que registra todos los movimientos financieros:

  • Debe — Montos que el cliente debe (ventas a cuenta, notas de débito)
  • Haber — Montos a favor del cliente (cobros, notas de crédito)
  • Saldo — Balance acumulado (positivo = deuda del cliente)

El historial muestra cada movimiento con fecha, comprobante, descripción, montos, y saldo resultante.

5.3 Cobranzas

Interfaz dedicada para registrar cobros a clientes con deuda pendiente.

  1. Seleccionar el cliente (se muestran solo los que tienen saldo > 0).
  2. Ingresar el monto a cobrar.
  3. Seleccionar medio de pago y cuenta bancaria (si es transferencia).
  4. Opcionalmente, distribuir el cobro entre varias facturas pendientes.
  5. Confirmar. El sistema registra el cobro en caja, actualiza el saldo, y genera el movimiento en cuenta corriente.

6.2 Presentaciones

Un producto puede tener múltiples presentaciones (variantes de empaque):

  • Unidad — Venta unitaria (1 unidad = 1 stock)
  • Caja x12, Caja x24, etc. — Paquete de N unidades con precio propio
  • Medio Cartón — Media caja (0.5 del cartón completo)

Cada presentación tiene: nombre, cantidad de unidades, SKU propio (opcional), costo y precio.

¿Cómo funciona el stock con presentaciones?
El stock se maneja en unidades base. Si un producto tiene 100 unidades y se vende una "Caja x24", el stock baja 24 unidades. Si se vende un "Medio Cartón" (cantidad 0.5 de un cartón de 12), el stock baja 6 unidades.

6.3 Combos

Los combos son productos compuestos por otros productos del catálogo.

  • Se define qué productos componen el combo y en qué cantidad
  • El stock del combo se calcula automáticamente según el componente con menor disponibilidad
  • El costo se calcula sumando los costos de los componentes
  • Al vender un combo, se descuenta el stock de cada componente

6.6 Marcas y categorías

  • Crear, editar y eliminar categorías y subcategorías
  • Marcar categorías como restringidas (no visibles en tienda online)
  • Ver cantidad de productos por marca/categoría

6.7 Lista de precios PDF

Genera un PDF descargable con los precios del catálogo, filtrable por categorías. Útil para enviar a clientes mayoristas.

7. Proveedores

Gestión completa de proveedores con datos fiscales y cuenta corriente.

Datos del proveedor

  • Básicos: Nombre, Razón Social, CUIT, Código, Condición IVA, Rubro
  • Contacto: Teléfono, Email, Web, Nombre de contacto
  • Dirección: Domicilio, Localidad, Provincia
  • Comercial: Plazo de pago, Días de entrega

Pestañas del proveedor

  • Resumen — Datos básicos y saldo pendiente
  • Compras — Historial de todas las compras realizadas
  • Cuenta Corriente — Movimientos con saldo actualizado

Pagos a proveedores

Se puede registrar un pago y distribuirlo entre varias facturas de compra pendientes. El sistema actualiza el saldo, registra en cuenta corriente, y opcionalmente genera un movimiento de egreso en caja.

8. Compras

8.1 Registro de compras

Registrar compras a proveedores con detalle de ítems, precios y montos.

  • Seleccionar proveedor y fecha
  • Agregar ítems del catálogo con cantidad y precio de compra
  • El total se calcula automáticamente
  • Registrar medio de pago y estado de pago

Estados de compra: Abierta Parcial Completa Cancelada

Estados de pago: Pagada Pendiente Pago Parcial

8.2 Pedidos a proveedores

Crear órdenes de compra previas a la recepción de mercadería. Permite hacer seguimiento del estado: pedido, en camino, recibido.

8.3 Reposición automática

El sistema analiza el stock y sugiere reposiciones basándose en:

  • Stock actual vs. mínimo — Detecta productos por debajo del umbral
  • Velocidad de venta — Calcula unidades vendidas por día
  • Días de stock restante — Estima cuántos días dura el stock actual
  • Cantidad sugerida — Recomienda cuánto pedir

Indicadores de nivel: Crítico Bajo OK

Desde esta vista se pueden crear órdenes de compra directamente agrupadas por proveedor.

9. Caja

Gestión completa del flujo de efectivo con sistema de turnos.

Sistema de turnos

  1. Abrir turno — Ingresar fecha, operador y monto inicial de efectivo. Se muestra un indicador verde en el sidebar mientras la caja está abierta.
  2. Registrar movimientos — Durante el día se registran ingresos y egresos. Las ventas y cobros se registran automáticamente.
  3. Cerrar turno — Ingresar el conteo final de efectivo. El sistema calcula la diferencia entre lo esperado y lo real.

Tipos de movimiento

TipoDescripciónEjemplo
IngresoDinero que entraVenta, cobro, depósito
EgresoDinero que salePago a proveedor, gasto, retiro
CancelaciónReversiónAnulación de venta

Medios de pago en caja

Cada movimiento registra el medio: Efectivo, Transferencia, Mixto, Cuenta Corriente. El resumen muestra totales separados por cada medio.

Cuentas bancarias

Se pueden configurar múltiples cuentas bancarias en Configuración → Pagos. Cada transferencia se asocia a una cuenta, permitiendo conciliación bancaria.

10. Stock

Ajustes de stock

Registrar correcciones al inventario fuera de ventas/compras normales:

Motivos de ajuste

  • Mercadería defectuosa
  • Mercadería vencida
  • Consumo interno
  • Venta al costo
  • Robo interno / externo
  • Diferencia de inventario (conteo físico)

Cada ajuste registra: producto, cantidad, costo unitario, motivo, y observación. Se genera un registro en stock_movimientos para auditoría.

Autoconsumo

Módulo para registrar el consumo interno de empleados (club de consumo):

  • Registrar miembros con nombre y PIN de seguridad
  • Registrar consumos por miembro (producto, cantidad)
  • Reportes mensuales por miembro y por producto

11. Reportes

Reporte de ventas

Resumen de ventas con filas expandibles que muestran los ítems de cada venta. Filtrar por día, mes, o rango de fechas. Incluye tipo de comprobante, cliente, total, y medio de pago.

Reporte de compras

Resumen de compras a proveedores con detalle de ítems. Mismos filtros que ventas.

Reporte de stock

Estado actual del inventario filtrable por categoría, subcategoría, y marca. Muestra código, nombre, stock, precio, costo, y capital invertido.

Resumen mensual

Informe completo de un mes con:

  • Total ventas, cantidad de ventas, notas de crédito
  • Total compras, costo de mercadería vendida
  • Ganancia neta (ventas - costos)
  • Top clientes y top productos
  • Desglose por medio de pago
  • Transferencias por cuenta bancaria
  • Detalle de egresos
  • Comparativa con el mes anterior

Ranking de clientes

Listado de clientes ordenados por volumen de compra. Permite identificar los mejores clientes.

Resumen por vendedor

Comisiones y rendimiento por vendedor. Muestra ventas totales, ventas comisionables (excluyendo categorías configuradas), y monto de comisión calculado.

Percepciones

Registro de percepciones impositivas (IIBB, IVA) sobre ventas, con jurisdicción, alícuota, base imponible, y monto calculado.

Exportación
Todos los reportes se pueden exportar a Excel para análisis externo.

12. Vendedores y comisiones

Gestión de representantes de ventas con cálculo automático de comisiones.

  • Datos: Nombre, email, porcentaje de comisión
  • Comisiones: Se calculan sobre las ventas del período seleccionado
  • Exclusiones: Se pueden excluir categorías del cálculo de comisión (ej: cigarrillos, bebidas)
  • Períodos: Filtrar por hoy, semana, mes, o rango personalizado

Cada vendedor se asigna a clientes, y las ventas de esos clientes generan comisión para el vendedor asignado.

14.3 Checkout

Proceso de compra paso a paso:

  1. Datos de contacto — Nombre, email, teléfono. Pre-rellenado si el cliente tiene cuenta.
  2. Método de entrega — Retiro en local o envío a domicilio.
  3. Dirección (si envío) — Seleccionar dirección guardada o agregar nueva.
  4. Fecha de entrega — Calendario con los días disponibles según configuración y zona del cliente.
  5. Medio de pago — Efectivo, Transferencia, o Mixto. Si hay recargo por transferencia, se muestra el monto adicional.
  6. Confirmar pedido — El sistema valida stock en tiempo real, calcula el total final, y genera el pedido.
Validación de stock en checkout
Antes de confirmar, el sistema re-verifica el stock de todos los productos. Si algún producto se agotó o tiene stock insuficiente, ofrece ajustar automáticamente las cantidades.

Montos mínimos

El carrito muestra una barra de progreso hacia el mínimo:

  • Naranja — Por debajo del mínimo de retiro
  • Azul — Habilitado para retiro, falta para envío
  • Verde — Habilitado para envío a domicilio

16. Despliegue y mantenimiento

Despliegue en VPS

El sistema se despliega en un VPS con un mecanismo de auto-deploy:

  1. Clonar el repositorio en el servidor: git clone [repo-url]
  2. Configurar las variables de entorno en el archivo .env
  3. Instalar dependencias: npm install
  4. Compilar: npm run build
  5. Iniciar con un process manager: pm2 start npm --name "enexpro" -- start

Auto-deploy con webhook

El sistema incluye un endpoint POST /api/pull que ejecuta git pull automáticamente al recibir una solicitud con el token correcto.

Configurar el webhook en GitHub/GitLab para que al hacer push a main, se envíe un POST a:

POST https://tu-dominio.com/api/pull
Header: x-pull-secret: [PULL_SECRET]

El servidor hace git pull origin main y se actualiza automáticamente.
